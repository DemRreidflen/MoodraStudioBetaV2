1. Учитывать заголовок как отдельный блок layout

Сейчас движок почти наверняка делает так:

pageHeight → заполняем текстом
потом рендерим заголовок

Это ошибка.

Заголовок должен участвовать в расчёте страницы.

Правильная логика:

pageHeight
↓
minus topMargin
↓
minus headerBlockHeight
↓
= доступная высота текста

Алгоритм:

availableTextHeight =
pageHeight
- topMargin
- bottomMargin
- headerHeight

Если страница начинается с заголовка:

headerHeight = measure(header)

Если нет:

headerHeight = 0

Только после этого начинается размещение строк.

2. Контроль висячих строк после заголовка

После заголовка нельзя оставлять одну строку текста.

Правило издательской верстки:

minLinesAfterHeading = 3

То есть:

ПЛОХО

Заголовок

строка текста
↓ page break

ПРАВИЛЬНО

Заголовок

строка
строка
строка

Если после заголовка помещается меньше 3 строк — заголовок переносится на следующую страницу.

Алгоритм:

if remainingHeightAfterHeader < (lineHeight * 3)
    moveHeaderToNextPage()
3. Исправить перенос абзацев между страницами

Сейчас система переносит абзац, но оставляет абзацный отступ.

Это ошибка.

Продолжение абзаца никогда не имеет first-line indent.

Правило:

paragraphStart = true → indent
paragraphStart = false → no indent

Алгоритм:

if paragraphContinuesOnNextPage
    firstLineIndent = 0
else
    firstLineIndent = paragraphIndent
4. Определение продолжения абзаца

Когда происходит page break:

paragraphSplit = true

Следующая страница должна начинаться так:

paragraph.isContinuation = true

И тогда:

text-indent: 0
5. Widow / orphan контроль внутри абзаца

Правильные параметры:

minLinesTop = 2
minLinesBottom = 2

То есть запрещено:

1 строка внизу страницы
1 строка вверху следующей

Алгоритм:

if linesRemainingInParagraph == 1
    moveLineToNextPage()
6. Правильная последовательность layout

Верстка страницы должна идти строго в таком порядке:

1 создать страницу
2 разместить header (если есть)
3 вычислить доступную высоту текста
4 размещать строки
5 проверять widow/orphan
6 переносить строки при необходимости
7 завершить страницу
7. Добавить защиту от ложных абзацев

Перед рендером строки нужно проверять:

if previousLine.belongsToSameParagraph
    indent = 0
8. Дополнительное правило издательств

Иногда используют hanging indent protection.

Если страница начинается с продолжения абзаца:

indent = 0
spacingBefore = 0

Это визуально склеивает абзац.

9. Финальная проверка страницы

После верстки страницы выполнить проверку:

noWidow
noOrphan
noIndentOnContinuation
headerHasMinimumLines

Если любое правило нарушено — пересобрать страницу.

💡 По твоим скринам видно ещё одну архитектурную вещь:
движок Moodra сейчас верстает страницу за страницей, а не анализирует абзац целиком.

Профессиональные движки (например в TeX или Adobe InDesign) делают наоборот — сначала разбивают абзац на строки, а потом распределяют по страницам.

Это сильно уменьшает:

висячие строки

пустоты

кривые переносы.
