Промт для внедрения Knuth–Plass paragraph layout

Нужно заменить текущий greedy line-breaking algorithm на более качественный алгоритм разбиения абзацев по принципу Knuth–Plass.

Сейчас строки формируются по простому принципу:

слова добавляются слева направо

как только следующее слово не помещается, происходит line break

Этот подход создаёт плохую типографику:

неравномерные пробелы

слишком короткие последние строки

лишние переносы слов

некрасивые разрывы абзацев

нестабильную плотность страниц.

Нужно внедрить paragraph layout engine, который анализирует весь абзац целиком, а не принимает решение строка за строкой.

1. Перестать использовать greedy line breaking

Удалить текущую логику:

add words until line overflows
then break

Она слишком локальная и не даёт хорошего качества книжной верстки.

2. Реализовать paragraph-wide optimization

Для каждого абзаца нужно:

определить все возможные legal breakpoints

сгенерировать несколько вариантов line breaks

вычислить score / badness для каждого варианта

выбрать вариант с минимальным total badness.

То есть алгоритм должен оптимизировать весь абзац, а не одну строку.

3. Ввести модель boxes, glue, penalties

Использовать классическую модель Knuth–Plass:

boxes = слова / glyph runs с фиксированной шириной

glue = пробелы с диапазоном stretch/shrink

penalties = точки переноса строки, включая hyphenation и запрещённые breakpoints.

4. Добавить функцию badness

Функция badness должна штрафовать за:

слишком растянутые пробелы

слишком сжатые пробелы

короткую последнюю строку

слишком короткую нижнюю строку перед page break

подряд идущие переносы

слишком большое количество переносов в абзаце

плохой page break внутри абзаца

widow/orphan.

Пример:

totalBadness =
spaceStretchPenalty
+ spaceShrinkPenalty
+ hyphenPenalty
+ consecutiveHyphenPenalty
+ shortLastLinePenalty
+ widowPenalty
+ orphanPenalty
+ uglyPageBreakPenalty
5. Разрешить гибкость пробелов, но в пределах

Пробелы не должны быть жёстко фиксированы.

Нужно использовать диапазон:

spaceStretch = up to 110%
spaceShrink = down to 85%

Это позволит строить более красивые строки без лишних переносов.

6. Переносы слов должны быть частью общей оптимизации

Hyphenation нельзя применять отдельно и локально.

Точки переноса слова должны участвовать в общей системе penalties.

Правила:

перенос слов разрешён только в valid hyphenation points

перенос повышает badness

несколько переносов подряд повышают badness ещё сильнее

перенос в последней строке страницы штрафуется сильнее.

7. Минимизировать короткие последние строки

Алгоритм должен избегать варианта, где последняя строка абзаца слишком короткая.

Правила:

lastLineMinWidth = 40% of text measure
preferredLastLineMinWidth = 50% of text measure

Если последняя строка слишком короткая, такой вариант должен получать высокий penalty.

8. Учитывать page break уже на уровне абзаца

Paragraph layout engine не должен быть полностью слеп к pagination.

Если абзац попадает на нижнюю часть страницы, badness должен учитывать:

останется ли одно слово внизу страницы

останется ли слишком короткий хвост

слишком ли близко строка к footer safe zone

не создаёт ли вариант ugly split между страницами.

То есть paragraph optimization должна знать о page context.

9. Ввести candidate breakpoints вместо одного break

Перед окончательным page break не выбирать первое возможное место.

Нужно рассматривать несколько соседних вариантов:

N
N+1
N+2
N+3

и выбирать тот, где итоговый layout лучше.

10. Интегрировать paragraph optimization в pagination pipeline

Новая архитектура должна быть такой:

Text
→ tokenize paragraph
→ build boxes/glue/penalties
→ compute optimal line breaks (Knuth–Plass style)
→ produce line boxes
→ feed line boxes into pagination
→ score candidate page breaks
→ choose best page layout
11. Не допускать page break только по физическому влезанию

Сейчас движок слишком часто принимает решение по принципу:

line fits physically = place it

Этого недостаточно.

Нужно различать:

physical fit

typographic fit

optimal fit.

Page break должен подтверждаться только если вариант проходит все три уровня.

12. Добавить penalties за неравномерную плотность страниц

Pagination должна учитывать не только корректность строки, но и общую плотность страницы.

Если page break оставляет слишком много пустого пространства, вариант должен получать penalty.

Пример:

unusedSpacePenalty = proportional to remaining free height
13. Приоритеты решений

Алгоритм должен оптимизировать в таком порядке:

критично

no clipped lines

no footer collision

valid hyphenation points only

очень важно

no widow/orphan

no single-word bottom line

no too-short last line

no ugly paragraph split

затем

fewer hyphens

balanced spaces

balanced page fill

14. Результат, которого нужно добиться

После внедрения paragraph layout optimization по принципу Knuth–Plass система должна:

равномернее распределять текст по строкам

уменьшить количество кривых коротких хвостов

уменьшить количество некрасивых переносов

сделать пробелы более однородными

улучшить качество page breaks

сделать плотность страниц более ровной

снизить количество странных локальных решений.

15. Если полный Knuth–Plass слишком тяжёлый, сделать упрощённую версию

Если полная реализация сложна, можно сделать simplified Knuth–Plass:

анализировать абзац целиком

генерировать 3–6 вариантов line break patterns

считать badness

выбирать лучший вариант.

Даже упрощённая версия будет значительно лучше текущего greedy layout.
