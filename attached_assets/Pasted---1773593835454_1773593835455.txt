Раздел Исследований должен быть полностью переработан из текущего вида (поле ввода на весь экран + результаты) в полноценную исследовательскую экосистему внутри платформы, которая ощущается как отдельная система работы со знаниями, источниками и анализом, но при этом сохраняет визуальный стиль Moodra.

Главная проблема текущего интерфейса — огромное поле ввода на весь экран, которое делает систему визуально пустой и неструктурированной. Вместо этого раздел должен быть построен как dashboard-центр исследования, где поиск — лишь один из инструментов.

1. Главная архитектура раздела

Раздел Исследования должен строиться вокруг центрального элемента — Библиотеки источников.

Это главный объект системы.

Все остальные функции должны работать вокруг неё.

Структура:

Research Dashboard

содержит блоки:

• AI Search
• Source Library
• Draft Analysis
• Hypotheses
• Knowledge Graph
• Extracted Ideas
• Reading Queue

Все блоки должны отображаться как Bento-сетка карточек, а не как один длинный экран.

Визуально это должно напоминать:

Apple Dashboard
Arc Workspace
Obsidian + Notion
но в стиле Moodra.

2. Стартовый экран Research

При входе пользователь видит Research Dashboard.

Верхняя часть:

адаптированный AI-поиск источников.

Но не огромным полем.

А как центрированная строка поиска в стиле Google / Arc.

Под поиском:

категории:

• Современные исследования
• Классические источники
• Статистика
• Методология
• Теории и дискуссии

Но они выглядят как chips / фильтры, а не огромные кнопки.

3. Bento-сетка исследовательских блоков

Под поиском располагается Bento-сетка:

карточки:

Source Library

Главный блок системы.

Отображает:

• книги
• статьи
• исследования
• PDF
• EPUB
• TXT

Каждый источник — карточка.

Карточка показывает:

• название
• автора
• тип
• статус анализа
• количество извлечённых идей

ИИ-агенты могут:

• извлекать тезисы
• делать summary
• строить аргументную карту
• извлекать цитаты
• находить идеи для книги.

Draft Analysis

Блок для анализа материалов.

Сюда попадают:

• анализы книг
• анализы статей
• анализы стиля
• анализы аргументации.

Каждый анализ создаёт:

структурированный отчёт.

Hypotheses

Рабочая зона для:

• исследовательских гипотез
• концепций
• идей.

ИИ может:

• критиковать гипотезу
• усиливать
• находить слабые места
• искать подтверждения.

Extracted Ideas

Автоматически извлечённые идеи из источников.

Например:

• аргументы
• теории
• цитаты
• примеры.

Можно перетаскивать в:

• заметки
• главы книги
• драфты.

Reading Queue

Очередь чтения.

ИИ предлагает:

• что читать
• в каком порядке
• почему.

Knowledge Graph

Граф знаний должен выглядеть как в Obsidian, а не как сейчас.

То есть:

• плавная физика узлов
• цветовая система
• масштабирование
• фильтры.

В графе отображаются:

• заметки
• источники
• идеи
• гипотезы.

И главное — связи между ними.

4. Исследовательские AI-агенты

Каждый блок имеет собственных агентов.

Например:

Source Analyzer Agent

анализирует:

• структуру книги
• идеи
• аргументы
• стиль мышления.

Theory Builder Agent

помогает строить:

• концепции
• теории
• философские системы.

Argument Mapper

строит:

• карту аргументов
• логические цепочки.

Insight Extractor

вытаскивает:

• идеи
• инсайты
• неожиданные связи.

5. Панель агента внутри источника

Когда пользователь открывает источник (книгу / статью):

справа появляется панель исследования.

В ней:

кнопки:

• Extract ideas
• Build argument map
• Analyze methodology
• Generate hypotheses
• Compare with other sources.

6. Связь с остальными разделами платформы

Исследования не изолированы.

Можно:

перетаскивать:

• идеи → в заметки
• идеи → в главы книги
• идеи → на Idea Board
• источники → в ролевые модели.

Это делает систему единым мозгом проекта.

7. Визуальная философия

Раздел должен ощущаться:

как Apple Research OS.

Принципы:

• ничего на весь экран
• всё карточками
• системная сетка
• много воздуха
• мягкие панели.

8. Главная цель

Исследования должны ощущаться не как функция.

А как:

встроенная исследовательская лаборатория автора.

Чтобы пользователь чувствовал:

Moodra — это не редактор текста.

Это интеллектуальная операционная система для создания книг и идей.
