Улучшение типографики страниц + предотвращение обрезанных строк + система технических страниц книги

В системе уже реализован Layout Mode для верстки книги. Нужно улучшить качество типографики, исправить поведение текста на страницах и добавить отдельный режим для создания технических страниц книги.

1. Равномерные отступы между текстом и номером страницы

Сейчас расстояние между текстом и номером страницы неравномерное.

Исправить:

расстояние от нижнего края текстового блока до номера страницы должно быть одинаковым на всех страницах

расстояние от номера страницы до нижнего края страницы также должно быть одинаковым

Структура страницы должна выглядеть так:

верхнее поле
↓
текстовая область
↓
равномерный отступ
↓
номер страницы
↓
нижнее поле

Это расстояние должно быть стабильным на всех страницах документа.

2. Исправить проблему «обрезанных строк»

Сейчас возникает ситуация, когда на странице видна половина строки текста (часть букв не помещается в текстовую область).

Это недопустимо для книжной верстки.

Исправить логику layout engine:

Если строка текста не помещается полностью в текстовую область страницы, то она должна целиком переноситься на следующую страницу.

Правило:

текст не должен обрезаться

строки должны отображаться полностью

если блок текста не помещается, он переносится на следующую страницу

Это касается:

обычного текста

заголовков

цитат

списков

3. Сохранение абзацного отступа во всех стилях книги

В системе есть несколько стилей оформления книги:

Classic

Modern

Vibe

Mono

Сейчас абзацный отступ работает только в некоторых режимах.

Исправить:

Во всех стилях оформления первый ряд каждого абзаца должен иметь абзацный отступ.

Этот параметр должен быть:

настраиваемым

доступным в панели Layout Settings

Настройка:

First line indent

Пользователь должен иметь возможность задать значение, например:

0.8em
1em
1.2em

Этот отступ применяется ко всем абзацам книги.

4. Удалить текущую титульную страницу

Сейчас титульная страница реализована в фиксированном виде и не подходит для гибкой верстки.

Нужно удалить текущую реализацию титульной страницы.

5. Добавить отдельный режим для создания технических страниц книги

В Layout Mode добавить новый раздел: Front Matter Builder.

Этот режим предназначен для создания служебных страниц книги, которые идут после обложки.

Структура:

Cover
↓
Title Page
↓
Copyright Page
↓
Dedication Page
↓
Book Content

Пользователь должен иметь возможность гибко настраивать эти страницы.

6. Конструктор титульной страницы (Title Page Builder)

Добавить гибкий редактор титульной страницы.

Пользователь должен иметь возможность добавлять элементы:

название книги

подзаголовок

имя автора

логотип издательства

год издания

декоративные элементы

иллюстрацию

Все элементы должны быть опциональными.

Если элемент не добавлен — он не отображается.

Также пользователь должен иметь возможность:

выбирать шрифт

менять размер текста

менять выравнивание

настраивать отступы между элементами

Пример структуры:

Book Title

Subtitle

Author Name

Publisher Logo

City
Year
7. Конструктор страницы копирайта (Copyright Page)

Добавить редактор страницы Copyright.

Это техническая страница, которая обычно находится на обороте титульной страницы.

Пользователь должен иметь возможность добавлять следующие поля:

Copyright ©

год издания

название издательства

ISBN

права на текст

аннотацию

редактора

дизайнера обложки

типографию

сайт издательства

Каждое поле должно быть:

редактируемым

опциональным

настраиваемым по стилю

Пользователь должен иметь возможность менять:

шрифт

размер текста

выравнивание

межстрочный интервал

отступы

8. Страница посвящения (Dedication Page)

Добавить возможность создать страницу посвящения.

Эта страница содержит короткий текст, например:

For my parents

или

Посвящается моей семье

Настройки:

выравнивание текста

шрифт

размер текста

вертикальное позиционирование на странице

9. Гибкость настройки технических страниц

Все технические страницы должны поддерживать:

выбор шрифта

размер текста

отступы

выравнивание

декоративные элементы

Эти страницы должны быть отдельной частью layout system, а не обычными страницами текста.

10. Итоговое поведение системы

После изменений Layout Mode должен:

корректно переносить текст между страницами

не показывать обрезанные строки

иметь равномерную структуру страниц

поддерживать абзацный отступ во всех стилях

иметь отдельный конструктор технических страниц книги

позволять гибко настраивать титульную страницу, копирайт и страницу посвящения.
