Промт для исправления страниц с заголовками и устранения висячих строк/слов

Нужно исправить две оставшиеся ошибки книжной верстки:

На страницах, где начинается глава и есть заголовок, текстовый блок рассчитывается неправильно. Заголовок визуально сдвигает текст вниз, но pagination engine не учитывает это как отдельный layout block. Из-за этого первая страница главы заполняется неправильно.

Движок допускает висячие строки / висячие слова / плохой разрыв абзаца между страницами: внизу страницы может остаться одно слово, слишком короткая последняя строка, либо предложение разрывается неаккуратно, а продолжение уходит на следующую страницу.

Нужно исправить это на уровне layout engine, а не косметически через CSS.

1. Ввести отдельный тип страницы для начала главы

Первая страница главы не должна обрабатываться как обычная текстовая страница.

Нужно создать отдельный page template:

ChapterStartPage

Структура страницы начала главы:

Page
 ├ TopMargin
 ├ ChapterHeaderBlock
 ├ GapAfterHeader
 ├ ContentArea
 └ Footer

Где:

ChapterHeaderBlock включает:

номер главы / small label

заголовок главы

декоративный разделитель, если он есть

GapAfterHeader — фиксированный отступ между заголовком и началом текста

ContentArea начинается только после полного измеренного header block.

2. Заголовок главы должен участвовать в расчёте высоты страницы

Сейчас движок, вероятно, сначала считает страницу как обычную, а потом сверху вставляет заголовок. Это ошибка.

Правильно:

availableTextHeight =
pageHeight
- topMargin
- bottomMargin
- footerHeight
- measuredChapterHeaderHeight
- gapAfterHeader

То есть заголовок главы должен быть полноценным layout-блоком, который уменьшает доступную высоту текста.

Нельзя делать так:

render text first
then visually push it down with header

Нужно делать так:

measure header first
then compute remaining text area
then paginate text into that reduced area
3. Реально измерять высоту блока заголовка

Высота заголовочного блока не должна быть захардкожена одной цифрой, если внутри меняются:

размер шрифта

межстрочный интервал заголовка

длина названия главы

наличие декоративного разделителя

расстояния между элементами блока.

Нужно измерять:

const headerHeight = chapterHeaderElement.getBoundingClientRect().height

или через внутреннюю typographic metrics систему.

И только после этого строить ContentArea.

4. Для страниц начала главы ввести отдельные правила заполнения

На первой странице главы нельзя пытаться заполнить страницу так же агрессивно, как обычную страницу, если это нарушает правила абзацев.

Но при этом нельзя оставлять нелогичные пустоты из-за того, что header block не учтён.

Нужно:

корректно пересчитать рабочую высоту

разрешить нормальный разрыв абзацев

не допускать висячих строк после заголовка.

5. Добавить правило minimum lines after heading

После заголовка главы нельзя оставлять слишком мало строк текста.

Правило:

minLinesAfterHeading = 3

Если после заголовка и gap помещается меньше 3 строк текста, заголовок должен переноситься на следующую страницу либо нужно перепагинировать начало главы.

Запрещено:

заголовок + 1 строка текста

заголовок + 2 строки текста, если дальше происходит кривой разрыв абзаца.

6. Устранить висячие строки / висячие слова / runts

Нужно добавить строгий контроль слишком коротких последних строк абзаца.

Запретить:

последнюю строку абзаца из 1 слова

последнюю строку абзаца, которая слишком короткая визуально

последнюю строку страницы, где осталось 1 слово или огрызок предложения.

Правило:

lastLineMinWidth = 30% of text measure

или минимум:

lastLineMinWords = 2

Если последняя строка абзаца на странице содержит только одно слово или слишком коротка, движок должен:

пересобрать line breaks внутри абзаца

попробовать другой перенос слова

перераспределить строки

если не помогает — перенести больше строк на следующую страницу.

7. Запретить “одно слово внизу страницы + пустая строка”

Сейчас на скрине видно сценарий, где внизу страницы остаётся одно слово, затем page break, а на следующей странице продолжается предложение. Это недопустимо.

Нужно запретить page break в таких случаях:

if lastVisibleLine.words <= 1
    rebalance paragraph

Также нужно запретить разрыв, если строка слишком короткая относительно measure:

if lastVisibleLine.width < 0.3 * textWidth
    rebalance paragraph
8. Разрыв абзаца между страницами должен быть умным

Разрыв абзаца разрешён, но только если соблюдены правила:

minLinesBottom = 2
minLinesTop = 2

То есть нельзя, чтобы:

внизу страницы осталась 1 строка абзаца

наверху следующей страницы осталась 1 строка продолжения абзаца.

Но этого недостаточно.

Нужно добавить ещё:

noSingleWordBottomLine = true
noShortRuntLine = true
9. Продолжение абзаца на следующей странице не должно иметь абзацный отступ

Если абзац продолжается после page break, первая строка продолжения не должна получать first-line indent.

Правило:

if paragraphContinuesOnNextPage
    firstLineIndent = 0
else
    firstLineIndent = paragraphIndent

Также не должно быть дополнительного spacing before.

10. Добавить paragraph balancing перед pagination

Перед тем как абзац раскладывается по страницам, нужно балансировать строки внутри абзаца.

Алгоритм должен:

минимизировать слишком короткие последние строки

минимизировать однословные строки

минимизировать резкий контраст длины строк

уменьшать вероятность плохого page break.

Идея:

paragraph
→ generate multiple legal line break variants
→ score each variant
→ choose variant with lowest badness

Штрафы в badness score:

one-word last line

too short last line

widow

orphan

bad page break after heading

excessive hyphenation.

11. Ввести penalties для плохих разрывов

Добавить систему штрафов:

widowPenalty = very high
orphanPenalty = very high
singleWordLinePenalty = very high
shortLastLinePenalty = high
pageBreakAfterSingleWordPenalty = critical
headingFollowedByTooLittleTextPenalty = critical

Движок должен избегать вариантов с такими штрафами.

12. Исправить page break после страниц с заголовком

На первой странице главы page break должен учитывать:

measuredChapterHeaderHeight

gapAfterHeader

widow/orphan control

no single-word line

paragraph continuation rules.

Нельзя рассчитывать начало главы как обычную страницу с тем же contentHeight, что и у остальных страниц.

13. Устранить ложные пустые строки перед переносом

Нужно проверить, не вставляет ли движок дополнительный spacer / blank line перед page break.

Запретить автоматическую вставку пустой строки, если она не является частью исходного текста.

Правило:

noArtificialBlankLineBeforePageBreak = true

Если внизу страницы появляется ощущение “одно слово и пустая строка”, движок должен проверять:

не был ли добавлен spacer

не сломался ли baseline step

не остался ли phantom line box.

14. Использовать разные page templates

Нужно разделить шаблоны страниц:

NormalTextPage
ChapterStartPage
TechnicalPage

У каждого шаблона свои правила:

разные header blocks

разная рабочая высота

разные ограничения на page fill.

Сейчас баг появляется потому, что ChapterStartPage визуально отличается, но логически обрабатывается почти как NormalTextPage.

15. Финальная проверка страницы перед commit

Перед сохранением результата page layout нужно проверить:

1. header height accounted for
2. no clipped lines
3. no widow
4. no orphan
5. no one-word bottom line
6. no too-short last line
7. no indent on paragraph continuation
8. no artificial blank line before page break

Если любое условие нарушено — страница должна быть пересобрана.

16. Итоговое поведение

После исправления система должна гарантировать:

страницы начала главы используют отдельный template

заголовок главы реально участвует в расчёте высоты страницы

текст на первой странице главы больше не сдвигается неправильно

не остаётся одно слово или слишком короткая строка внизу страницы

предложение не разрывается некрасиво между страницами

continuation paragraph на следующей странице начинается без абзацного отступа

widow/orphan control работает и на обычных страницах, и на страницах с заголовками.
