Полный промт для исправления обрезания строк

Исправить алгоритм книжной верстки так, чтобы обрезание строк стало физически невозможным.

Сейчас движок иногда размещает строку, даже если она частично выходит за пределы текстового блока. Это недопустимо для книжной верстки.

Система должна работать только на основе реальных измерений высоты строк, а не приблизительных расчетов.

1. Полностью отказаться от estimation-системы

Сейчас система, вероятно, рассчитывает количество строк примерно.

Нужно удалить любой код вида:

estimatedLinesPerPage
approxLineHeight

И заменить на измерение реального DOM-размера.

Правильная логика:

lineHeight = element.getBoundingClientRect().height
2. Строка добавляется только после проверки

Перед добавлением строки необходимо проверять:

currentHeight + lineHeight <= textAreaHeight

Если условие false, строка не может быть размещена.

Правильная логика:

if currentHeight + lineHeight > textAreaHeight
    startNewPage()
else
    placeLine()

Это делает обрезание невозможным на уровне движка.

3. Жесткая граница текстового блока

У каждой страницы должен быть строго определенный текстовый контейнер.

page
 └ textArea

Высота textArea фиксированная.

textAreaHeight = pageHeight - topMargin - bottomMargin

Никакой текст никогда не может выходить за границу textArea.

Также нужно включить защиту:

overflow: hidden

Но главное — не допускать переполнения логикой, а не CSS.

4. Добавить финальную проверку страницы

После завершения верстки страницы нужно выполнить проверку:

if lastLine.bottom > textArea.bottom
    moveLastLineToNextPage()

Это страховка, если произошла ошибка расчёта.

5. Использовать baseline grid

Профессиональные издательские системы используют baseline grid, чтобы строки всегда идеально совпадали.

Алгоритм:

baseline = lineHeight
linePosition = baseline * lineIndex

Тогда строки никогда не наползут друг на друга.

6. Убрать возможность двигать номер страницы

Позиция номера страницы должна быть фиксированной.

Запретить изменение вертикального отступа.

pageNumberPosition = bottomMargin / 2

UI-контрол для изменения позиции нужно удалить.

7. Финальный алгоритм верстки страницы

Правильная логика должна выглядеть так:

createPage()

for paragraph in chapter:

    for line in paragraph:

        measure(lineHeight)

        if currentHeight + lineHeight <= textAreaHeight
            placeLine()
            currentHeight += lineHeight

        else
            if widowOrphanViolation
                movePreviousLineToNextPage()

            startNewPage()
            placeLine()
            currentHeight = lineHeight
