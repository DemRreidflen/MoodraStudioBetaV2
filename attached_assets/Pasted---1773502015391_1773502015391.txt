Промт для выравнивания нижнего уровня страниц и исправления рваных переносов

Нужно исправить две оставшиеся системные ошибки layout engine:

На обычных текстовых страницах нижний уровень текстового блока слишком сильно гуляет. На одних страницах остаётся много пустого пространства снизу, на других текст доходит почти до нормального уровня. Для книжной верстки это выглядит нестабильно.

Переносы слов местами выглядят рваными, неаккуратными и типографически слабыми.

Нужно исправить это на уровне layout engine, а не стилями.

1. Ввести единый вертикальный ритм страницы

Для всех обычных текстовых страниц одного шаблона нужно использовать единый baseline grid и единый target fill level.

Структура:

все обычные страницы книги используют один и тот же ContentAreaHeight

строки привязаны к baseline grid

page break выбирается так, чтобы страницы заканчивались примерно на одном вертикальном уровне.

Нужно добавить правило:

normalTextPages should converge to a consistent bottom text line level

То есть у обычных страниц не должен хаотично гулять нижний край текста.

2. Ввести target page fill band вместо случайного заполнения

Сейчас движок, похоже, допускает слишком широкий диапазон заполнения страницы.

Нужно ввести целевой диапазон заполнения:

targetPageFill = 92% to 97% of ContentArea

Для обычных текстовых страниц page break должен стремиться к этому диапазону.

Если страница заполняется, например, только на 80–85%, такой вариант должен получать высокий penalty, если это не вызвано более важными правилами.

3. Добавить penalty за нестабильный нижний край текста

Движок должен штрафовать варианты layout, в которых соседние обычные страницы заканчиваются на слишком разном уровне.

Пример:

bottomAlignmentPenalty = abs(currentPageBottomFill - targetBottomFill)

Или ещё лучше — сравнивать страницу с нормой шаблона:

bottomRhythmPenalty = distance from preferred bottom baseline

Чем дальше последняя строка страницы от предпочтительного нижнего уровня, тем хуже вариант.

4. Разделить страницы по типам

Нужно считать отдельно:

NormalTextPage

ChapterStartPage

TechnicalPage

Требование одинакового нижнего уровня применяется в первую очередь к:

NormalTextPage

Не нужно пытаться выровнять по одному уровню страницы начала главы, содержание, копирайт и другие специальные шаблоны.

Но обычные страницы одной главы/книги должны выглядеть ритмично и равномерно.

5. Не делать ранний page break без крайней необходимости

Если на странице остаётся слишком много пустого пространства снизу, движок обязан попробовать:

добавить ещё одну строку

пересобрать line breaks текущего абзаца

изменить точки переноса слов

выбрать другой breakpoint.

Нельзя завершать страницу рано, если ещё можно получить более плотную и более ровную страницу без нарушения типографики.

6. Использовать candidate breakpoints, а не один break

Перед page break система должна рассматривать несколько соседних кандидатов:

N
N+1
N+2
N+3

И выбирать не первый физически возможный вариант, а тот, который лучше по совокупному score:

нет клиппинга

нет widow/orphan

нет ugly split

нормальный нижний уровень страницы

меньше пустого пространства

меньше плохих переносов.

7. Исправить рваные переносы слов

Сейчас переносы выглядят слишком рваными, потому что движок, вероятно, делает их слишком локально.

Нужно добавить строгие правила:

7.1 Перенос только по словарю языка

Использовать только valid hyphenation points:

hyphenationPoints = dictionaryBased(language)

Нельзя разрывать слово в визуально случайном месте.

7.2 Перенос всегда должен отображаться типографически корректно

Если слово переносится между строками, на конце строки обязательно должен быть дефис.

if wordSplitsAcrossLines:
    renderHyphen = true

Нельзя допускать “молчаливый разрыв” слова.

7.3 Ограничить количество переносов

Нужно задать пределы:

maxHyphensPerPage = 3
maxConsecutiveHyphenatedLines = 2

Если лимит превышается, алгоритм обязан искать другой line break pattern.

7.4 Ввести штраф за ugly hyphenation

Некоторые переносы формально допустимы, но визуально слабые.

Нужно штрафовать:

переносы в нижней строке страницы

переносы, создающие слишком короткий хвост

переносы, после которых на новой строке или странице остаётся визуально слабое продолжение.

Пример:

uglyHyphenPenalty = high
bottomLineHyphenPenalty = higher
pageEdgeHyphenPenalty = very high
8. Интегрировать переносы в paragraph optimization

Переносы нельзя выбирать отдельно от всего абзаца.

Paragraph engine должен:

строить несколько вариантов разбиения абзаца

учитывать пробелы

учитывать переносы

учитывать длину последней строки

выбирать лучший вариант целиком.

То есть hyphenation должна быть частью общей paragraph optimization, а не последним костылём.

9. Добавить penalty за “рваную” страницу

Если страница выглядит визуально рыхло — слишком много пустого пространства снизу, слишком ранний break, слабый хвост абзаца — такой вариант должен штрафоваться.

Пример совокупного score:

totalScore =
unusedSpacePenalty
+ bottomRhythmPenalty
+ widowPenalty
+ orphanPenalty
+ uglyParagraphSplitPenalty
+ hyphenPenalty
+ uglyHyphenPenalty
+ bottomLineQualityPenalty

Выбирать нужно вариант с наименьшим totalScore.

10. Ввести preferred bottom baseline

Для обычных текстовых страниц нужно определить предпочтительный нижний уровень окончания текста:

preferredBottomBaseline = last usable baseline before footer safe zone

Цель движка: стараться завершать обычные страницы как можно ближе к этой линии, не нарушая других правил.

Это создаст ощущение книжной ритмичности и профессиональной верстки.

11. Не выравнивать страницы искусственно через растяжение абзацев

Очень важно:

нельзя решать проблему разного нижнего уровня через искусственное растягивание межстрочного интервала или через вертикальное растяжение абзацев на странице.

Запрещено:

увеличивать line-height от страницы к странице

растягивать paragraph spacing ради заполнения страницы

вручную тянуть текст вниз.

Нужно решать это только через:

правильный line breaking

правильный paragraph balancing

правильный выбор page break.

12. Пост-валидация обычной страницы

После построения обычной текстовой страницы выполнить проверку:

validateNormalTextPage():
    noClippedLines
    noFooterCollision
    noWidow
    noOrphan
    noUglyParagraphSplit
    noBadHyphenation
    fillWithinPreferredBand
    bottomCloseToPreferredBaseline

Если вариант не проходит — пересобрать страницу.

13. Приоритеты правил

Применять правила в таком порядке:

критично

no clipped lines

no footer collision

valid hyphenation only

очень важно

no widow/orphan

no ugly paragraph split

no broken word without hyphen

важно

minimize ugly hyphens

keep page fill within target band

keep bottom level consistent across normal text pages

14. Итоговое поведение

После исправления система должна гарантировать:

обычные текстовые страницы заканчиваются примерно на одном нижнем уровне

снизу не остаётся случайно большого пустого пространства

page break не происходит слишком рано

переносы слов выглядят типографически нормально

нет рваных или визуально слабых переносов

страницы одного шаблона выглядят как единая книжная сетка, а не как случайный набор контейнеров.
