Полный промт для полной пересборки системы верстки Moodra

Удалить текущую систему верстки книги и полностью пересобрать layout-движок с нуля по профессиональной издательской архитектуре.

Текущая система использует приблизительную (estimation) модель расчета страниц и нестабильную pagination-логику. Это приводит к множеству ошибок:

обрезанные строки

текст, пересекающий номер страницы

пустые области страницы

неправильные переносы

висячие строки

некорректное завершение абзацев.

Новая система должна соответствовать принципам профессиональной книжной верстки, аналогичным тем, которые используются в издательских системах.

1. Архитектура Layout Engine

Создать двухфазный layout-движок:

Text
↓
Line Layout
↓
Paragraph Balancing
↓
Pagination
↓
Page Render

Страницы не должны строиться напрямую из текста.

2. Фаза 1 — Line Layout (формирование строк)

На первом этапе текст разбивается на абзацы и строки.

Структура:

Document
 └ Paragraph
     └ Lines

Каждый абзац должен быть преобразован в массив строк.

Алгоритм:

взять текст абзаца

вычислить доступную ширину строки

применить переносы слов

учитывать kerning и spacing

сформировать line boxes.

Результат:

paragraph → [line1, line2, line3, line4]

Каждая строка должна иметь реальные метрики:

width

height

baseline

ascent

descent.

Для расчета использовать реальные typographic metrics DOM / canvas, а не приблизительные оценки.

3. Фаза 2 — Paragraph Balancing

После формирования строк абзаца нужно применить алгоритм балансировки строк.

Цель:

сделать строки абзаца визуально равномерными.

Проблема обычной верстки:

Очень длинная строка
Очень длинная строка
Очень длинная строка
короткая

Это выглядит плохо.

Алгоритм balancing должен:

перераспределять переносы слов

регулировать переносы внутри абзаца

минимизировать слишком короткие строки.

Правила:

избегать последней строки короче 40% ширины

избегать резкого контраста длины строк

уменьшать количество переносов.

При балансировке можно:

менять места переноса слов

менять line breaks.

4. Фаза 3 — Pagination (разбиение на страницы)

После балансировки строки распределяются по страницам.

Алгоритм:

for each line:
    if line.bottom <= ContentAreaBottom:
        place line
    else:
        create new page
        place line

Контент может размещаться только внутри ContentArea.

5. Структура страницы

Каждая страница должна иметь фиксированную структуру:

Page
 ├ Header
 ├ ContentArea
 └ Footer
Header

может содержать название главы.

ContentArea

основная зона текста.

Правила:

строки не могут выходить за границы

строки не могут обрезаться

строка должна переноситься только целиком.

Footer

фиксированная область.

Содержит номер страницы.

Контент не может пересекать footer.

6. Номер страницы

Номер страницы должен:

располагаться в footer

иметь фиксированное расстояние от нижнего края страницы

пользователь не может менять это расстояние

номер страницы нельзя перемещать.

Удалить горизонтальную линию (divider) над номером страницы.

7. Правила типографики

Добавить профессиональные правила верстки.

Widow / Orphan control

Запретить:

одну строку абзаца в начале страницы

одну строку абзаца в конце страницы.

Минимум:

2 строки сверху
2 строки снизу

Если правило нарушается — переносить строки на следующую страницу.

Hanging punctuation

Запретить строки, заканчивающиеся:

запятой

короткими предлогами.

Строка должна переноситься корректно.

Висячие строки

Запретить одиночные короткие строки в конце страницы.

8. Hyphenation (переносы слов)

Добавить автоматический перенос слов.

Правила:

перенос только по правилам языка

перенос обозначается дефисом

в следующей строке продолжается слово.

Пример:

иннова-
ционная

Ограничения:

максимум 2–3 переноса подряд

избегать избыточных переносов

если возможно — переносить строку без разрыва слова.

9. Заполнение страницы

Страница должна заполняться максимально.

Не допускается:

большие пустые области

ранний перенос абзацев.

Контент должен доходить максимально близко к нижней границе ContentArea.

10. Запрет обрезания строк

Добавить строгую проверку:

if line.bottom > ContentAreaBottom:
    move line to next page

Строка никогда не должна:

обрезаться

частично попадать в footer.

11. Режим редактирования текста

В режиме написания текста:

запретить:

изменение размера шрифта

изменение межстрочного интервала

изменение типографики.

Редактор должен работать как простой текстовый редактор, аналогично Notion.

Все параметры типографики настраиваются только в режиме верстки книги.

12. Технические страницы книги

Технические страницы должны иметь отдельные шаблоны.

Титульная страница

На титульной странице запрещены:

аннотация

copyright

служебные тексты.

Разрешено только:

название книги

автор.

Страница аннотации

Структура:

АННОТАЦИЯ
[текст аннотации]

Правила:

текст начинается с абзаца

блок имеет ограниченную область

количество символов ограничено

текст не может выходить за границы блока.

Название должно автоматически переводиться в зависимости от языка книги.

Страница copyright

Структура:

Аннотация

[текст]

Copyright
[блок информации]

Блок copyright должен находиться у нижнего края страницы.

Внутри блока:

год

правообладатель

дизайнер обложки

редактор

издатель.

Блок должен иметь фиксированное расстояние от нижнего края страницы.

13. Локализация

Системные слова должны автоматически переводиться:

Annotation

Copyright

Author

Cover designer

Editor.

Перевод должен зависеть от языка книги.

14. Ограничение структуры

Пользователь не может:

ломать структуру технических страниц

перемещать служебные блоки

менять layout страниц.

Разрешается только редактирование текста внутри блоков.

15. Проверка перед рендером

Перед созданием страницы система должна проверять:

нет ли обрезанных строк

нет ли висячих строк

не пересекается ли текст с footer

корректны ли переносы

соблюден ли widow/orphan контроль.

Если правило нарушено — пересчитать pagination.

16. Итоговая цель

Создать стабильный профессиональный layout-движок для книг, который формирует страницы по правилам издательской типографики.

Страницы должны выглядеть как в профессиональной книжной верстке и не содержать ошибок:

обрезанных строк

висячих строк

пустых областей

неправильных переносов

пересечения текста с номером страницы.
