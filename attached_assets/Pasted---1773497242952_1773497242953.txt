Полный промт для внедрения профессиональной типографики

Необходимо внедрить профессиональную систему типографики для книжной верстки, аналогичную той, что используется в профессиональных издательских движках.

Система должна решать следующие задачи:

правильные переносы слов

равномерное выравнивание текста

отсутствие «дырок» между словами

отсутствие висячих строк

отсутствие обрезаний

оптическое выравнивание текста.

Все алгоритмы должны работать до этапа финальной pagination.

1. Добавить систему переносов слов

При выравнивании по ширине текст должен использовать автоматические переносы слов.

Если слово не помещается в строку полностью, система должна:

проверить допустимые точки переноса

разделить слово

поставить дефис.

Пример:

ПРАВИЛЬНО

инфор-
мация

НЕПРАВИЛЬНО

инфо
рмация
Алгоритм переносов

Использовать словари переносов для разных языков.

hyphenationDictionary(language)

Каждое слово анализируется на допустимые точки:

in-for-ma-tion

Система выбирает самую позднюю точку переноса, которая позволяет строке поместиться.

Ограничения переносов

Запретить перенос если:

слово короче 5 символов

остаётся менее 2 символов до дефиса

остаётся менее 3 символов после дефиса.

Пример запрета:

ре-
ка
Ограничение количества переносов

Чтобы текст выглядел аккуратно:

maxHyphensPerParagraph = 3
maxHyphensInRow = 2
2. Улучшить выравнивание по ширине

Сейчас текст, скорее всего, просто растягивает пробелы.

Это создаёт дырки между словами.

Нужно внедрить умное распределение пробелов.

Допустимые значения
spaceStretch = 110%
spaceShrink = 85%

То есть пробелы могут:

увеличиваться максимум на 10%

уменьшаться максимум на 15%.

3. Оптическое выравнивание (micro-typography)

Некоторые символы должны немного выходить за край строки, чтобы текст выглядел ровным.

Это называется optical margin alignment.

Пример символов:

.
,
:
;
-
"
'

Они должны выступать на 5–10% за край текстового блока.

4. Улучшить алгоритм построения строки

Система не должна формировать строки последовательно.

Нужно использовать алгоритм оптимизации абзаца.

Абзац должен анализироваться целиком, чтобы найти лучший вариант разбиения строк.

Это тот же принцип, который использует TeX.

Логика
analyze entire paragraph

calculate multiple line break options

score each layout

choose lowest badness score
Функция badness

Штраф начисляется за:

слишком растянутые пробелы

слишком сжатые пробелы

переносы

висячие строки.

Пример:

badness =
spaceStretchPenalty +
hyphenPenalty +
widowPenalty

Выбирается вариант с минимальным значением.

5. Улучшить widow/orphan контроль

Добавить строгие правила.

Запрещено:

1 строка абзаца внизу страницы

1 строка абзаца вверху следующей страницы.

Минимум:

minLinesTop = 2
minLinesBottom = 2
6. Добавить контроль последней строки абзаца

Последняя строка абзаца не должна быть слишком короткой.

Правило:

lastLineWidth >= 35% textWidth

Если строка короче, алгоритм должен пересобрать абзац.

7. Улучшить переносы внутри страниц

При переносе абзаца между страницами нужно:

проверить widow/orphan

перераспределить строки.

Пример:

если на странице остаётся только одна строка абзаца — перенести две строки.

8. Добавить контроль ритма текста

Все строки должны быть привязаны к baseline grid.

baselineGrid = lineHeight

Это делает вертикальный ритм идеальным.

9. Проверка перед финальной версткой страницы

Перед финальным размещением страницы выполнить проверки:

noClippedLines
noWidows
noOrphans
spaceStretch <= limit
hyphenLimit <= allowed

Если правило нарушено — пересобрать страницу.
