В приложении уже реализованы два режима:

Text Editor Mode — основной режим написания текста (Notion-style редактор)

Layout Mode — режим визуальной верстки книги с предпросмотром страниц и экспортом PDF

Нужно исправить несколько критических UX и layout проблем и улучшить редактор текста.

1. BUG: Layout Mode постоянно возвращается на первую страницу

Сейчас при любом изменении текста Layout Mode пересчитывает страницы и возвращает пользователя на титульную страницу.

Это неправильное поведение.

Исправить:

Layout Engine должен сохранять текущую страницу, на которой находится пользователь.

После перерасчёта страниц Layout Mode должен оставаться на той же странице.

Если структура изменилась, прокрутка должна автоматически находить ближайший соответствующий блок.

Pseudo logic:

save currentPageIndex
re-render layout
restore scroll to currentPageIndex
2. BUG: титульная страница растягивается в режиме разворота

В режиме Book Spread (две страницы) титульная страница сейчас растягивается на двойной размер.

Это неправильно.

Правильное поведение:

Первая страница книги всегда одиночная

Далее страницы идут парами

Пример:

Page 1

Page 2 | Page 3
Page 4 | Page 5
3. BUG: страницы визуально растягиваются

Если глава длинная, сейчас страница растягивается по высоте.

Это ломает концепцию книжной верстки.

Исправить:

каждая страница должна иметь фиксированный размер

текст должен перетекать на следующую страницу

CSS пример:

.book-page {
  width: 148mm;
  height: 210mm;
  overflow: hidden;
}

Использовать paged layout правильно.

4. BUG: формат страницы не совпадает при экспорте

Сейчас при экспорте PDF:

игнорируется выбранный формат (A5 / A4)

игнорируются margins

игнорируется spacing

Исправить pipeline:

Layout Settings
→ CSS paged media
→ HTML render
→ Puppeteer
→ PDF

PDF должен полностью повторять layout preview.

PDF export должен учитывать:

page size

margins

typography

spacing

page numbers

headers/footers

5. BUG: форматирование текста не применяется

На странице редактора текста сейчас:

изменение размера шрифта не работает

изменение шрифта не работает

Исправить:

toolbar должен менять реальные стили текста

изменения должны сохраняться в документе

6. Глобальная смена шрифта книги

Добавить настройку:

Book Font

При выборе шрифта он применяется ко всему документу.

Pipeline:

Font selector
→ update document CSS
→ update editor
→ update layout preview

Шрифт должен обновляться во всех страницах сразу.

7. Отступ первой строки абзаца

Добавить кнопку:

First Line Indent

Она применяет:

text-indent: 1.2em;

Это стандарт книжной типографики.

8. Улучшение редактора текста

Text Editor Mode — основной режим работы автора.

Layout Mode используется только для визуальной проверки верстки.

Поэтому редактор текста должен стать мощнее.

Добавить функции:

текст

bold
italic
underline
strikethrough

заголовки

H1 — chapter
H2 — section
H3 — subsection

выравнивание

left
center
right
justify

списки

bullet list
numbered list

отступы

increase indent
decrease indent

spacing

line spacing
paragraph spacing

9. Книжные элементы

Добавить специальные блоки:

Chapter title

Epigraph

Quote

Footnote

Page break

Footnote должен:

автоматически нумероваться

отображаться внизу страницы

10. Page Break UX

Page break должен:

вставляться через /pagebreak

отображаться как линия

──────── Page Break ────────

И влиять на layout.

11. Улучшение Layout Mode

Layout Mode должен показывать реальные страницы книги.

Добавить:

Single Page Mode
Book Spread Mode

Также добавить:

Zoom
Page navigation

12. Главное UX правило

Text Editor Mode — главная среда написания книги

Layout Mode — визуальная проверка верстки

Поэтому:

редактор должен быть максимально функциональным

layout должен точно отображать будущий PDF

Итог

После исправлений система должна:

корректно отображать страницы

не растягивать их

сохранять текущую страницу при редактировании

экспортировать PDF точно как preview

иметь полноценный редактор текста для написания книг.
