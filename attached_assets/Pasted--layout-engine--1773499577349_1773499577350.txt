Нужно исправить оставшуюся ошибку layout engine: движок всё ещё допускает тупые висячие строки и плохие разрывы абзацев между страницами.

Сейчас проблема выглядит так:

внизу страницы остаётся слишком короткая последняя строка абзаца

иногда это буквально огрызок строки

остальная часть того же абзаца продолжается на следующей странице

формально overflow нет, но визуально книжная верстка выглядит неправильно

Это значит, что текущий pagination engine проверяет только физическое влезание строк по высоте, но не оценивает качество последней строки перед page break.

Нужно исправить это на уровне paragraph balancing и page-break rules.

1. Запретить page break после “плохой последней строки”

Перед page break система должна проверять качество последней строки абзаца, которая остаётся внизу страницы.

Нельзя оставлять внизу страницы:

строку из одного слова

слишком короткую строку

визуально слабый хвост абзаца

строку, после которой остальная часть того же абзаца продолжается на следующей странице слишком большим куском

Добавить правило:

forbidBadBottomRuntLine = true
2. Ввести проверку минимальной длины последней строки перед разрывом страницы

Нужно оценивать последнюю строку абзаца перед page break по двум критериям:

по количеству слов
minBottomLineWords = 2
по визуальной ширине строки
minBottomLineWidth = 35% of text measure

Если нижняя строка не проходит хотя бы один из критериев, page break в этом месте запрещён.

Пример:

если внизу страницы осталось 1 слово → запрещено

если строка очень короткая визуально → запрещено

если хвост абзаца выглядит как огрызок → запрещено

3. Перед page break делать paragraph rebalance, а не просто перенос

Сейчас движок, вероятно, делает так:

line fits → place line
next line does not fit → new page

Этого недостаточно.

Нужно добавить промежуточный этап:

before committing page break:
    evaluate bottom paragraph fragment
    if fragment quality is bad:
        rebalance current paragraph

То есть перед финальным page break система обязана попробовать:

переразбить строки текущего абзаца

сдвинуть line breaks внутри абзаца

изменить переносы слов

при необходимости перенести на следующую страницу не одну строку, а больше.

4. Добавить paragraph balancing внутри абзаца

Нужно, чтобы движок не считал строки абзаца фиксированными навсегда.

Он должен уметь пересобирать абзац, если page break создаёт плохой хвост.

Алгоритм:

paragraph
→ generate multiple legal line-break variants
→ score each variant
→ choose the variant that minimizes:
    - widow
    - orphan
    - one-word line
    - short bottom line
    - ugly page break
5. Ввести penalties для плохих хвостов

Добавить систему штрафов для layout engine.

Пример:

widowPenalty = 1000
orphanPenalty = 1000
singleWordBottomLinePenalty = 1500
tooShortBottomLinePenalty = 1200
badParagraphSplitPenalty = 1500

Если page break даёт:

одно слово внизу страницы

слишком короткую строку

плохой разрыв абзаца

движок должен искать другой вариант layout.

6. Если плохой хвост нельзя исправить line breaks, переносить больше строк

Если paragraph balancing не может улучшить последнюю строку, движок должен переносить на следующую страницу не только одну строку, а целый фрагмент абзаца.

Пример логики:

if bottomFragmentIsBad:
    moveOneMoreLineToNextPage()
    re-evaluate
if still bad:
    moveAnotherLineToNextPage()

Пока не будет достигнут нормальный разрыв.

7. Ввести minimum bottom fragment quality

Последний фрагмент абзаца на странице должен быть “достаточно полноценным”.

Правило:

bottomParagraphFragmentMinLines = 2

Но этого мало.

Также нужно:

bottomFragmentMustNotEndWithShortRunt = true

То есть внизу страницы должен оставаться либо:

нормальный кусок абзаца

либо разрыв должен происходить раньше

Но нельзя оставлять внизу страницы визуально жалкий хвост абзаца.

8. Разрыв абзаца должен учитывать остаток на следующей странице

Система должна оценивать не только то, что осталось внизу текущей страницы, но и то, что перейдёт наверх следующей.

Плохой вариант:

внизу страницы короткий хвост

на следующей странице огромный остаток абзаца

Это означает, что page break поставлен не в оптимальном месте.

Добавить правило:

page break must optimize both:
- current page bottom fragment
- next page top fragment
9. Не делать page break сразу после слишком короткой строки

Если последняя строка текущей страницы короткая, page break должен быть запрещён, даже если следующая строка уже не помещается по высоте.

Нужно не просто проверять:

nextLineFits?

а проверять:

isCurrentBottomLineGoodEnoughForBreak?

Если нет — пересобирать абзац.

10. Исправить алгоритм page break decision

Нужен такой порядок:

1. измерить следующую строку
2. понять, помещается ли она физически
3. если не помещается:
    3.1 оценить последнюю строку текущей страницы
    3.2 оценить верхний фрагмент следующей страницы
    3.3 попробовать rebalance paragraph
    3.4 попробовать alternative line breaks
    3.5 только потом подтверждать page break
11. Ввести правило no ugly paragraph split

Добавить флаг:

noUglyParagraphSplit = true

Page break внутри абзаца запрещён, если:

последняя строка текущей страницы слишком короткая

первая строка продолжения слишком одинокая

визуальный разрыв выглядит неестественно.

12. Продолжение абзаца на следующей странице

Если абзац продолжается на следующей странице:

первая строка продолжения не должна иметь first-line indent

не должно быть дополнительного spacing before

continuation должно выглядеть как естественное продолжение абзаца.

Правило:

if paragraphContinuesOnNextPage:
    firstLineIndent = 0
    spacingBefore = 0
13. Проверка страниц с заголовками тоже должна учитывать bad bottom line

Для страниц начала главы уже есть другой template, но на них тоже нужно применять те же правила:

no widow

no orphan

no short bottom line

no one-word bottom line

no ugly paragraph split.

То есть ChapterStartPage не должен использовать упрощённую pagination-логику.

14. Добавить финальную post-layout validation

После построения страницы выполнять обязательную проверку:

validatePage():
    noClippedLines
    noWidow
    noOrphan
    noSingleWordBottomLine
    noTooShortBottomLine
    noUglyParagraphSplit
    noIndentOnContinuedParagraph

Если хотя бы одно правило нарушено — страница пересобирается.

15. Добавить специальный режим strict book typography

Включить для книжной верстки строгий режим:

strictBookTypography = true

Он должен включать:

widow/orphan control

paragraph balancing

no single-word line

no short bottom line

no ugly paragraph split

no indent on continued paragraph

header-aware pagination.

16. Итоговое поведение

После исправления система должна гарантировать:

внизу страницы не остаётся тупой короткий хвост абзаца

внизу страницы не остаётся одна короткая строка или одно слово

page break внутри абзаца происходит только в типографически нормальном месте

continuation paragraph на следующей странице выглядит естественно

страницы с заголовками и обычные страницы подчиняются одинаково строгим правилам качества разрыва.

17. Короткая техформулировка для реализации

Нужно не просто контролировать overflow, а оптимизировать качество разрыва абзаца при pagination.

Ключевая проблема:
текущий engine умеет не обрезать строки, но всё ещё не умеет выбирать хороший breakpoint внутри абзаца.

Нужно добавить:

paragraph balancing

bad bottom line detection

no ugly paragraph split

penalties for short runt lines

re-layout before committing page break.
