Твоя задача — внедрить интеллектуальную систему проверки текста и автокоррекции (как в Scrivener), но адаптированную для веб-приложения.

ФУНКЦИОНАЛ ДОЛЖЕН РАБОТАТЬ НА 4 ЯЗЫКАХ:

русский (ru)

немецкий (de)

украинский (uk)

английский (en)

🎯 ЦЕЛЬ

Создать 3-уровневую систему:

Мгновенное автоисправление очевидных опечаток (без UI)

Умные предложения (через API LanguageTool)

Нативная браузерная проверка (spellcheck)

🧠 ОБЩАЯ АРХИТЕКТУРА

Frontend:

Отслеживает ввод текста (input / textarea / contenteditable)

Делает debounce (300–600ms)

Анализирует последнее слово и последнее предложение

Отправляет текст на backend API для проверки

Backend:

Принимает текст и язык

Делает запрос к LanguageTool API

Возвращает список ошибок и предложений

1️⃣ УРОВЕНЬ: БАЗОВЫЙ SPELLCHECK (FRONTEND)

Для текстового редактора:

включить:
spellCheck={true}
lang="auto" (или конкретный язык документа)

добавить поддержку определения языка:
пользователь может выбрать язык документа:
"ru", "de", "uk", "en"

язык сохраняется в состоянии документа

2️⃣ УРОВЕНЬ: АВТОКОРРЕКЦИЯ ОПЕЧАТОК (FRONTEND)

Реализовать систему автозамены для ОЧЕВИДНЫХ ошибок.

ЛОГИКА:

отслеживать последнее введённое слово

при вводе пробела / точки / Enter:
→ анализировать слово

использовать:

словарь (простая структура JSON)

расстояние Левенштейна

частотные ошибки

ПРИМЕРЫ:

"ошибысяч" → "ошибся"
"несногуо" → "немного"
"teh" → "the"

УСЛОВИЯ АВТОЗАМЕНЫ:

заменять ТОЛЬКО если:

расстояние ≤ 2

слово не найдено в словаре

есть 1 очевидный кандидат

confidence высокий

иначе → НЕ менять

3️⃣ УРОВЕНЬ: УМНАЯ ПРОВЕРКА (LanguageTool API)
Backend endpoint:

POST /api/text-check

BODY:
{
text: string,
language: "ru" | "de" | "uk" | "en"
}

Backend логика:

использовать публичный API:
https://api.languagetool.org/v2/check

отправлять:
text
language

возвращать:
matches:

message

replacements

offset

length

Frontend:

подсвечивать ошибки:

underline (тонкое, не красное жирное)

при клике:

показывать popover

список замен

4️⃣ UI / UX

Реализовать:

Подсветка:

мягкое подчёркивание

не агрессивное (не как Word)

Popover:

при клике на ошибку:

список вариантов

кнопка "применить"

кнопка "игнорировать"

Настройки пользователя:

Добавить настройки:

язык документа

режим проверки:

OFF

BASIC (только spellcheck)

SMART (spellcheck + suggestions)

AUTO (включая автоисправление)

5️⃣ PERFORMANCE

debounce: 300–600ms

не отправлять весь текст:
только последнее предложение или блок

кешировать результаты

6️⃣ БЕЗОПАСНОСТЬ

НЕ отправлять API ключи в frontend

backend работает как proxy

7️⃣ ДОПОЛНИТЕЛЬНО (ВАЖНО)

НЕ менять текст автоматически, если:

это художественный текст

есть нестандартные слова

низкая уверенность

добавить whitelist слов пользователя

8️⃣ РАСШИРЕНИЕ В БУДУЩЕМ

подготовить архитектуру под:

self-hosted LanguageTool

кастомные словари

AI rewrite (опционально)

🧩 РЕЗУЛЬТАТ

После внедрения пользователь должен:

писать с автоисправлением как в Scrivener

видеть мягкие подсказки

не терять свой стиль

работать на 4 языках без переключения инструментов

Реализация должна быть модульной:

frontend hooks / services

backend service layer
