Мы улучшили pagination engine, но сейчас появились новые проблемы:

страницы имеют разную плотность текста

иногда остаётся слишком много пустого пространства внизу

иногда абзац переносится слишком рано

иногда на следующую страницу переносится слишком маленький кусок текста.

Это происходит потому, что система принимает решение о page break слишком рано и не сравнивает несколько вариантов layout.

Нужно внедрить систему Page Density Balancing и Layout Scoring, как в профессиональных системах верстки.

1. Генерировать несколько вариантов page break

Вместо одного варианта разрыва страницы движок должен рассматривать несколько возможных точек.

Алгоритм:

candidateBreakpoints = [
line N
line N+1
line N+2
line N+3
]

То есть движок должен попробовать несколько мест разрыва, а не первое возможное.

2. Ввести оценку плотности страницы

Каждый вариант страницы должен получать score.

Пример:

score =
unusedSpacePenalty
+ orphanPenalty
+ widowPenalty
+ hyphenPenalty
+ paragraphSplitPenalty

И выбирается вариант с наименьшим штрафом.

3. Ограничить пустое пространство страницы

Страница не должна заканчиваться слишком высоко.

Правило:

minPageFill = 85%

То есть контент должен заполнять минимум 85% области текста.

Если заполнение меньше — нужно попробовать другой break.

4. Запретить перенос слишком маленького хвоста абзаца

Нельзя переносить на новую страницу:

1 строку абзаца

2 очень короткие строки

маленький кусок текста.

Правило:

minLinesNextPage = 3

Если на следующей странице остаётся меньше строк, нужно перенести больше текста.

5. Балансировать соседние страницы

Если страница заканчивается слишком пусто, движок должен попробовать:

добавить ещё одну строку

добавить ещё один перенос

переразбить предыдущий абзац.

То есть система должна подгонять границу страниц, чтобы плотность была похожей.

6. Ввести penalty за пустую страницу

Если страница заканчивается с большим пустым пространством:

unusedSpacePenalty = remainingHeight * weight

Это заставит движок искать более плотный вариант.

7. Снизить страх переносов

Сейчас система слишком избегает переносов слов.

Но переносы — это нормальный инструмент верстки.

Правило:

maxHyphensPerPage = 4

До этого числа переносы допустимы.

8. Не переносить полслова на новую страницу

Если перенос слова происходит:

на последней строке страницы

и продолжение слова будет первым словом новой страницы

то такой перенос нужно избегать.

Лучше:

перенести всё слово

или перенести строку.

9. Балансировать разворот

Поскольку редактор показывает развороты, желательно учитывать баланс двух страниц.

Если левая страница пустая, а правая переполнена — движок должен попробовать другой break.

10. Финальный алгоритм

Новый алгоритм pagination должен быть:

layoutParagraphs()

for each page:

    generateCandidateBreakpoints()

    for each breakpoint:
        simulateLayout()
        calculateScore()

    chooseLayoutWithLowestScore()

Это позволит:

уменьшить пустые зоны

сделать страницы более равномерными

убрать странные переносы.

Важный момент

То что сейчас происходит у тебя — нормальная стадия разработки layout engine.

Все системы через это проходят:

сначала строки обрезаются

потом появляются висячие строки

потом пустые страницы

потом баланс страниц.

Даже TeX проходил через это.
